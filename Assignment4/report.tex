\section{Introduction}

In this assignment, we investigate the use of recurrent neural networks (RNNs) for stock market prediction. Unlike traditional feedforward neural networks, recurrent models are specifically designed to process sequential data by maintaining a hidden state that evolves over time. This makes them particularly well-suited for time series forecasting tasks such as predicting stock prices.

We evaluate three models:
\begin{itemize}
    \item Basic RNN (baseline)
    \item Gated Recurrent Unit (GRU)
    \item Long Short-Term Memory (LSTM)
\end{itemize}

We use historical stock data from four companies: AAPL, GOOG, AMD, and NVDA. The goal is to predict the next day's closing price based on the previous 50 days. All datasets were normalized using MinMax scaling and split into 80\% training and 20\% testing data.


\section{Task 2}

The initial phase of the project involved establishing a baseline using a standard Recurrent Neural Network. Unlike traditional feedforward networks, RNNs process sequences by maintaining a hidden state $h_t$ that acts as a memory of previous inputs. The state update is governed by the following recurrence:
$$
h_t = \tanh(W_{hh}h_{t-1} + W_{xh}x_t + b_h)
$$


While mathematically sound, vanilla RNNs often struggle with "Vanishing Gradients," where the signal from the beginning of the 50-day window diminishes by the time it reaches the output layer. To combat this and ensure stable convergence, we implemented several structural optimizations:
\begin{itemize}
    \item \textbf{Stacked Layers:} Using two layers allows the network to learn higher-level abstractions of price movements (e.g., the stacked layers allow the model to learn hierarchical temporal representations).
    \item \textbf{Dropout (0.2):} Randomly deactivating neurons during training prevents the model from overfitting to specific patterns in the training data, forcing it to learn generalized market patterns.
    \item \textbf{Batch Normalization:} Applied to the final hidden state $h_n$ to re-center and re-scale the data, preventing activations from becoming too small or too large before the final prediction.
\end{itemize}

The model consists of:
\begin{itemize}
\item Input size: 1 (closing price)
\item Hidden size: 64
\item Number of layers: 2
\item Dropout: 0.2
\item Batch normalization on final hidden state
\item Fully connected output layer
\end{itemize}

The model takes a sequence of 50 previous stock prices and predicts the next value. The hidden state from the final time step is passed through batch normalization and then into a linear layer. The intuition behind this structure is that the RNN captures temporal dependencies in stock prices, while batch normalization helps stabilize training and improve convergence.

\section{Task 4}

The GRU simplifies the gating process by using an update gate and a reset gate. This allows the model to selectively discard information that is no longer relevant to the current price trend.
$$
    z_t = \sigma(W_z \cdot [h_{t-1}, x_t]) \quad (\text{Update Gate})
$$
This gating mechanism allows the GRU to control how much past information is retained or forgotten, helping mitigate the vanishing gradient problem observed in standard RNNs.

The architecture remains similar to the baseline:
\begin{itemize}
    \item 2 GRU layers
    \item Hidden size: 64
    \item Dropout: 0.2
    \item Batch normalization
    \item Fully connected output layer
\end{itemize}

\section{Task 5}

The LSTM is our most sophisticated model. It introduces the \textbf{Cell State ($c_t$)}, a dedicated memory lane that is regulated by three distinct gates: Forget, Input, and Output. This allows the model to preserve relevant long-term dependencies within the input sequence more effectively than standard RNNs or GRUs.

The architecture includes:
\begin{itemize}
    \item 2 LSTM layers
    \item Hidden size: 64
    \item Dropout: 0.2
    \item Batch normalization
    \item Fully connected output layer
\end{itemize}

\section{Discussion}
After extensive testing and optimization, several key conclusions can be drawn regarding the use of RNNs for financial forecasting. A general trend between model complexity and performance can be observed, although it is not strictly consistent across all stocks. While the Basic RNN is computationally efficient (avg. 7.8s training time), it is more prone to errors when the stock price deviates from its current momentum. The LSTM, while taking longer to train (avg. 9.6s), provides a much "smoother" prediction curve that is less reactive to daily noise and more focused on the underlying trend.

\begin{itemize}
    \item \textbf{RNN:} Best for steady-growth stocks with high autocorrelation (e.g., AAPL).
    \item \textbf{GRU:} An excellent middle-ground, often training faster than the LSTM while offering significantly better protection against noise than the RNN.
    \item \textbf{LSTM:} The superior choice for high-volatility technology stocks like NVDA and GOOG, where long-term historical context is necessary to filter out short-term market "shocks."
\end{itemize}

The implementation of Dropout and Batch Normalization proved essential in stabilizing the models. These techniques helped reduce overfitting and improved generalization to the test data. Without these, the LSTM tended to overfit on the training data. By enforcing these constraints, the final models achieved an average test accuracy exceeding 96.5\%, demonstrating that recurrent architectures are highly capable of modeling the complex dynamics of the stock market when properly optimized. These results highlight the importance of model selection and hyperparameter tuning in time series prediction tasks.
